Edit Registry Dialog

This dialog implements a registry editor which allows to view, change or delete the currently selected items.
 

 
Please note that modifications of keys or values are made permanent if you select another line inside the dialog. The Cancel button will cancel the current edit operation, but it will NOT undo insertions, deletions or changes done so far!
 
The editor is capable to edit registry entries of type REG_SZ (+REG_EXPAND_SZ), REG_MULTI_SZ, REG_DWORD and REG_BINARY. Items the editor cannot handle are displayed as '?' and cannot be edited of course.
 
Another point to mention is that comparing Win9x with WinNT platforms there are some slight differences regarding registry functions. On Win9x systems there always exists a default value when calling RegQueryValueEx, but not when enumerating values with RegEnumValue function. This is not the case on WinNT/2K/XP systems.
 
To avoid fatal damage you won't be allowed to delete entries CLSID, Typelib, Interface or AppId.
 
What is shown in the editor depends on the view it has been started from. The additional keys listed below will be included only if they exist, of course.
  • CLSID View: referenced TypeLib, ProgId, VersionIndependantProgId and AppId entries will be included. This is especially useful if a CLSID shall be removed from registry. But be careful with these type of actions. The recommended way to remove a CLSID is to unregister the server. Removing CLSIDs manually should only be considered if the binary implementing the class is no longer available or is unable to unregister itself.
  • Interface View: referenced TypeLib and CLSID (in ProxyStubClsid32 subkey) keys will be included.
  • AppId view: Key in value AppId will be included.
  • HKCR view: referenced CLSID key will be included. If entry is a file extension, referenced ProgId in default value will be added.
