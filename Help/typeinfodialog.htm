Type Information Dialog

This tabbed dialog shows a type information entry of a type library.
 

 
A type information may include up to 3 tables, which are displayed by this dialog:
  • Functions: Normally type informations of kind TKIND_DISPATCH, TKIND_INTERFACE and TKIND_MODULE will include a table of functions. Columns are:
    • Name: Name of function returned by ITypeInfo::GetNames.
    • memid: Function's member ID.
    • FuncKind, InvKind, CallConv: FuncKind specifies if function is virtual, static or dispatch-only. InvKind (Invocation kind) indicates if this is a property function and if so, what kind. CallConv is function's calling convention.
    • rcType: Type of return value.
    • Params: Names and types of parameters. Double-clicking an item will show parameters in more detail (in, out, optional).
    • Flags: Function's flags. A translatíon should appear in right pane of status line for current item. Values are restricted, source, bindable, requestedit, displaybind, defaultbind, hidden, usesgetlasterror, defaultcollelem, uidefault, nonbrowsable, replaceable, immediatebind.
    • ofsVft/Entry: For virtual functions specifies offset in vtable. For static functions displays dll name and entry name/ordinal.
  • Variables: Type informations of kind TKIND_ENUM, TKIND_RECORD, TKIND_UNION and sometimes TKIND_MODULE will most likely include a table of constants/variables. Columns are:
    • Name: Name of constant/variable returned by ITypeInfo::GetNames.
    • memid: member ID.
    • varkind: Specifies if item is a constant, static variable, member of a type or to access via IDispatch::Invoke only.
    • Type: Type of constant/variable.
    • Value/Offset: Shows value (for constants) or offset within instance (for member variables).
    • Flags: Variable's flags. Values are readonly(1), source(2), bindable(4), requestedit(8), displaybind(0x10), defaultbind(0x20), hidden(0x40), restricted(0x80), defaultcollelem(0x100), uidefault(0x200), nonbrowsable(0x400), replaceable(0x800), immediatebind(0x1000).
  • Interfaces: TKIND_COCLASS type informations will include interfaces. Type informations of kind TKIND_DISPATCH and TKIND_INTERFACE will display here the interface they are inherited from. Note that for type informations of kind TKIND_DISPATCH there will exist an entry here pointing to a TKIND_INTERFACE type information with identical IID. Only that entry will be used by COMView for ASM include file generation.
Right-clicking an item will open a context menu with commands:
  • View Item: displays a simple dialog which shows current item in more detail.
  • View ref. TypeInfo: referenced type information will be shown in another typeinfo dialog.
  • Copy: copies selected lines to the clipboard.
  • Help: if available displays context sensitive help.
