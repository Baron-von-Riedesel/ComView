Other Features of COMView

  • Command Line Arguments: COMView will accept the following:
     
    • COMView filename: filename must include a type library resource which can be loaded with LoadTypLib. If that's true, a type library dialog will be opened.
    • COMView /B </OUT:output directory> filename: this will cause COMView to run in batch mode. In this mode COMView will generate ASM include files without any user dialog. With optional parameter /OUT: directory for files to be generated may be set.
    • COMView /S filename: this will cause COMView to handle file as if option Default action is Bind to File is set.

     
  • Drag'n Drop: COMView registers as drop source and target. Default action depends on option Default action is Bind to File. Drag with right mouse button to let COMView show a context menu to select one of these commands:
    • Show a type library dialog,
    • Register Server
    • Unregister Server
    • Bind to a File
    • Create Link
    • Open Storage
    • Open Stream
    If a path exists for an item in CLSID or typelib view, this item may be dragged. Only clipboard format CF_HDROP is supported.

    •  
  • DDE Support: COMView supports DDE communication, application string is COMView, topic string should be NULL or system and command which will be recognized is open, for example [open("C:\TEMP\AsmCtrl.TLB")]. One possibility of using DDE is to set DDEEXEC key for file extensions in registry, which may avoid to start multiple instances of an application.
