Log Window

This window will display
  • internal logs about object creation/deletion
  • "incoming" events, that is, COMView being called by hosted objects
The "standard" way to get events from a server in COM is through interface IConnectionPointContainer, which must be exposed by the object. If this interface is available, the hosting application may "connect" to the object and will then be notified of certain events occuring. Therefore to see events in the Log window you will have to:
  • create an object exposing interface IConnectionPointContainer
  • in the object dialog right-click an entry in the list of outgoing interfaces and select menu item Connect
  • open Log window if not already open. Incoming events will only be logged if this window is open.
With option Log to debugger window all messages may be routed to a debugger. Furthermore there exist some switches in object options to influence the amount of messages displayed. So for example it may be interesting to show the control's queries of ambient properties or other methods of the OLE container. And COMView may be instructed to log all calls to its IUnknown::QueryInterface methods.
