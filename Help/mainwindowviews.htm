Main Window Views

COMView's main window will show one of several views selectable by tabs at the bottom of the window.

CLSID View

 CLSID view lists subkeys of registry entry HKEY_CLASSES_ROOT\CLSID. Normally these subkeys are CLSIDs. Standard columns are
  • Column Text is default value of CLSID subkey.
  • For Columns Type and Type Value see description of option CLSID Type Search Order in CLSID options.
  • Columns ProgId and TypeLib are subkeys of CLSID entry.
A userdefined column is available for this view and will allow to easily recognize entries which are 'Controls', 'Automation Objects', 'Embeddable Objects' or whatever. To select a value for userdefined column from a predefined set right-click header or open CLSID options.

TypeLib View

 TypeLib view lists subkeys of registry entry HKEY_CLASSES_ROOT\TypeLib. But that is not a 1:1 view of this subkey. COMView will display each version and each locale of a type library in its own line. So you may see lines with identical GUIDs in this view.

Interface View

 Interface view lists subkeys of registry entry HKEY_CLASSES_ROOT\Interface. Remember that COMView will only know interface IDs listed here if it scans an object for exposed interfaces.

AppId View

 AppId view will list subkeys of registry entry HKEY_CLASSES_ROOT\AppId.

Component Category View

 This view will list subkeys of registry entry HKEY_CLASSES_ROOT\Component Categories.

HKCR View

 HKCR view will list subkeys of registry root entry HKEY_CLASSES_ROOT. A userdefined column is available for this view and may be set by right-clicking header or in HKCR options.

Created Objects View

 This view will show objects created by comview (with menu items Create Instance and Create Instance On for example). Edit submennu will therefore be changed and have some additional commands:
  • View Object:: opens an object dialog for this object.
  • Unlock Object:: will unlock a previously locked object. If not used otherwise, object may be destroyed and will disappear afterwards.
  • View Moniker:: will show the moniker itself in an object dialog. Command will be enabled if a moniker exists, which is the case for example if object has been created by file menu command Bind to File.

ROT View

 This view will show entries in Running Object Table. Applications register itself there by RegisterActiveObject and unregister by RevokeActiveObject. Edit submennu will have additional command:
  • View Object:: opens an object dialog for this object.
