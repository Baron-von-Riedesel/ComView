Parameter Dialog

This dialog will be opened from inside properties dialog if property/method requires parameters.
 

 
The combobox on the right of each parameter input field may allow some parameter type tuning. For most parameter types it will show entries VT_EMPTY and VT_BSTR only. That's no problem since COMView will try to convert a BSTR to the expected type. Exceptions from this are parameters of type:
  • VT_VARIANT: In this case you may specify the type of the variant by selecting the proper type in the combobox.
  • VT_DISPATCH: possible values are VT_EMPTY and VT_DISPATCH. The only way to enter a value for such a parameter is to use function Paste Object.
  • VT_UNKNOWN: possible values are VT_EMPTY and VT_UNKNOWN. The only way to enter a value for such a parameter is to use function Paste Object.
For optional parameters COMView will translate VT_EMPTY to VT_ERROR and set scode to DISP_E_PARAMNOTFOUND, which is interpreted as parameter not supplied.

Button commands

  • Browse: will open a dialog to select a filename which is copied to current parameter field.
  • Paste Object: This command will be enabled only if an object has been copied to the clipboard in object dialog and type of current parameter is VT_UNKNOWN or VT_DISPATCH.

Hints

  • To enter a number in hexadecimal use prefix "&H" as in VB.
  • Parameter declared as [out] will be displayed in the Log window.
